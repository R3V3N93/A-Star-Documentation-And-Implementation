해당 가이드는 실제 실행 가능한 코드를 제시하는 대신, 의사코드를 사용하여 개념을 설명을 할 예정이다. 의사코드는 대체로 C#의 문법을 따르며, 가독성을 지키기 위하여 실제 C#과 어긋나는 문법이 있음을 유의 바란다.

이하에서 소개되는 의사 클래스들은 의사코드에서 추상적으로 제시되는 클래스들의 함수, 변수의 기능을 주석과 의사코드를 사용해 명시한 것으로, 지금 읽지 않고(그럴 필요도 없고), 나중에 다시 찾아와 어떤 함수들이 어떤 역할을 하는지 알아가면 된다.
###### 노드 클래스
```C#
Class Node
{
	public Node[] Neighbors() {} // 해당 노드와 선분으로 연결된 모든 이웃 노드. 
	public int    Cost()      {} // 해당 노드의 비용
								 // 실제로는 노드와 노드 간의 선분의 비용을 일컬어야 하지만, 
								 // 설명의 편의성을 위하여 노드 자체에 비용을 할당함.
}

const Node START_NODE       = 알고리즘의_시작_노드;
```

###### Queue, PriorityQueue
```Csharp
// <T>는 템플릿으로, 해당 클래스가 입력된 타입 T를 기반으로 전개된다는 뜻이다.
// 예를 들어, Queue<int>는 정수형 숫자를 기반으로 한 동작을 수행할 것이다.
Class Queue<T>
{
	List<T> items; // 데이터들이 저장된 리스트.
	public void Put(T item) 
	{
		this.items.Add(item); // item을 items의 뒤에서 부터 추가함.
		return;
	}
	public T Get() 
	{
		T firstItem = this.items[0]; // 첫번째 원소를 따로 저장하고,
		this.items.RemoveAt(0); // 첫번째 원소를 제거. 나머지 원소는 앞당겨짐.
		return firstItem; // 저장한 첫번째 원소를 반환
	}
	public int Size()
	{
		return items.Count(); // 현재 Queue의 크기 반환
	}
}
```
