향후 이어지는 알고리즘들에서 사용되는 가장 기본적인 개념은 '==경계==(*frontier*)'이다. 경계는 시작 노드를 기점으로 모든 방향으로 끝없이 뻗어나가는 고리로, 그래프 전체를 우선적으로 탐색하는데 사용된다. 경계가 그래프 전체를 탐색하는 과정은 다음과 같다:

1. 가장 초기의 경계는 시작 노드와 이웃한 모든 노드를 포함한다.
2. 경계에서 노드 하나를 고른다.
3. 고른 노드와 이웃한 모든 노드를 탐색하여, 해당 노드가 도달할 수 있는 노드라면, 경계에 포함시킨다.
4. 고른 노드와 탐색된 노드는 모두 '도착한 노드' 목록에 포함시킨다.
5. 그래프의 모든 노드를 탐색할 때 까지 2~4번 과정을 반복한다.

(파이선 문법을 약간 빌린) 의사 코드는 다음과 같다:
```python
# 경계 큐 할당
경계 = new Queue()
경계.Add(시작 노드)
# 도착한 노드 리스트 할당
도착 = new List()
도착.Add(시작 노드)


while (경계.Count() > 0):
	고른노드 = 경계.Get() # 경계 큐의 첫번째 원소를 고름
	for (이웃 in 고른노드.Neighbors()): # 고른 노드의 이웃 노드 중에서
		if (도착.Find(이웃) == false): # 도착하지 않은 노드가 있다면
			경계.Add(이웃) # 그 노드에서 부터 다시 이웃 탐색 하기 위해 경계에 추가
			도착.Add(이웃) # 그 노드에 도착!
	
```
<코드 1>
향후 이어지는 알고리즘들은 모두 이 로직을 따라 갈 예정이다. 의사 코드를 분석해봤다면 눈치챘겠지만, 이 로직은 도달할 수 있는 노드만을 탐색할 뿐, 실제로 최단 거리를 탐색하지는 않는다. 경로를 탐색하기 위하여, 위 로직을 살짝 변형하겠다.

```python
# 경계 큐 할당
경계 = new Queue()
경계.Add(시작 노드)

# 경로 노드 리스트 할당
# 한 노드에서 어느 노드로 도달했는가를 저장하는 사전형
경로 = new dict<노드, 노드>() # {경로[B] == A}는 A->B를 뜻한다.
경로[시작 노드] = null

while (경계.Count() > 0):
	고른노드 = 경계.Get() # 경계 큐의 첫번째 원소를 고름
	for (이웃 in 고른노드.Neighbors()): # 고른 노드의 이웃 노드 중에서
		if (경로.Find(이웃) == false): # 도착하지 않은 노드가 있다면
			경계.Add(이웃) # 그 노드에서 부터 다시 이웃 탐색 하기 위해 경계에 추가
			경로[이웃] = 고른노드 # 그 노드에 도착! 경로를 저장함
	
```

해당 그림에서 알 수 있다시피, 우리가 원하는 것과 달리, 화살표가 시작 노드에서 목적노드를 향하는 것이 아닌, 반대로 목적 노드에서 시작 노드를 향하고 있다. 이는 우리가 할당한 경로 사전형이 '다음 노드'를 저장하는 것이 아닌 '이전 노드'를 저장하기 떄문이다. 그렇다면 우리가 원하는 경로로 만들려면 어떻게 해야할까? 간단하다. 그냥 화살표를 역으로 따라가면 된다.

```python
현재노드 = 목적지 노드
최단경로 = new List() 
while (현재노드 != 시작노드):
	최단경로.Add(현재노드)
	# 경로[]를 기반으로 역으로 따라간다.
	현재노드 = 경로[현재노드] # 헷갈리지 말자! 경로[]는 이전 노드를 가리킨다.

# 실제 경로의 노드 개수가 시작 노드와 목적지를 포함하여 n+2개 있다고 해보자.
# 루프가 끝난 후, 최단경로에는 다음과 같이 저장되어 있다:
# 최단경로 = [목적지 노드, 노드1, 노드2, ..., 노드n]
# 시작 노드는 없다! 시작 노드를 넣기 전에 루프가 끝났기 때문.

현재노드.Add(시작노드) # 마지막으로 시작 노드를 최단경로에 추가하고,
최단경로.Reverse() # 최단경로를 뒤집으면 우리가 원하는 경로 완성!
```
