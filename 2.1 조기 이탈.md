지금까지 학습한 너비 우선 탐색 알고리즘의 작동 과정은 다음과 같다:
1. 모든 노드에 대하여 역순의 경로를 탐색한다.
2. 목적지에서 탐색한 역순의 경로를 따라 시작 지점까지의 최단 경로를 구한다.
3. 역순의 경로를 뒤집어 정방향 경로로 만든다.

첫번째 단계에서 '모든'이라는 키워드에 주목해보자. 당연하겠지만, 모든 경로를 탐색하면, 그에 따라 불필요한 탐색이 발생할 수 밖에 없다. 그렇다면 목적지를 찾자마자 노드 탐색 루프에서 '조기 이탈'할 수는 없을까?

이 그림을 보면 한 눈에 알 수 있다. 조기 이탈을 한다면 원래 탐색하던 노드 양의 50~60%가 된다. 그렇다면 조기이탈은 어떻게 구현할 수 있을까? 간단하다. 챕터 2의 <코드 2>에 두 줄만 추가하면 된다.

```python
# 경계 큐 할당
경계 = new Queue()
경계.Add(시작 노드)

# 경로 노드 리스트 할당
# 한 노드에서 어느 노드로 도달했는가를 저장하는 사전형
경로 = new dict<노드, 노드>() # {경로[B] == A}는 A->B를 뜻한다.
경로[시작 노드] = null

while (경계.Count() > 0):
	고른노드 = 경계.Get() # 경계 큐의 첫번째 원소를 고름
	
	if 고른노드 == 목적지 노드: # 목적지를 진작에 찾았다면
		break # 조기 이탈!
	
	for (이웃 in 고른노드.Neighbors()): # 고른 노드의 이웃 노드 중에서
		if (경로.Find(이웃) == false): # 도착하지 않은 노드가 있다면
			경계.Add(이웃) # 그 노드에서 부터 다시 이웃 탐색 하기 위해 경계에 추가
			경로[이웃] = 고른노드 # 그 노드에 도착! 경로를 저장함
	
```

지금 소개된 조기 이탈의 사용법은 가장 대표적이고 간단한 용법이다. 조기 이탈은 이탈하는 조건에 따라 무궁무진하게 활용 될 수 있다. 예를 들어 시작 노드에서 총 n노드를 이동할 수 있는 방법의 경우의 수를 구할 수도 있고, n개의 목적지를 찾을 수도 있다.
