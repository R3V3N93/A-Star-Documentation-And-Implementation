지금까지 학습한 너비 우선 탐색 알고리즘의 작동 과정은 다음과 같다:
1. 모든 노드에 대하여 역순의 경로를 탐색한다.
2. 목적지에서 탐색한 역순의 경로를 따라 시작 지점까지의 최단 경로를 구한다.
3. 역순의 경로를 뒤집어 정방향 경로로 만든다.

첫번째 단계에서 '모든'이라는 키워드에 주목해보자. 당연하겠지만, 모든 경로를 탐색하면, 그에 따라 불필요한 탐색이 발생할 수 밖에 없다. 그렇다면 목적지를 찾자마자 노드 탐색 루프에서 '***조기 이탈****(*early exit*)'할 수는 없을까?

![](https://github.com/R3V3N93/A-Star-Documentation-And-Implementation/blob/main/Pasted%20image%2020260321025142.png)
이 그림을 보면 한 눈에 알 수 있다. 이 그림의 환경에서, 조기 이탈을 한다면 원래 탐색하던 노드 양의 50~60%가 된다. 그렇다면 조기이탈은 어떻게 구현할 수 있을까? 간단하다. 챕터 2의 <코드 2>에 두 줄만 추가하면 된다.

```C#
// 고리 큐 할당
Queue<Node> frontier = new Queue();
frontier.Add(START_NODE);
// 경로 노드 리스트 할당
// 한 노드에서 어느 노드로 도달했는가를 저장하는 사전형
Dictionary<Node, Node> came_from = new Dictionary<Node, Node>(); // {came_from[B] == A}는 A->B를 뜻한다.
came_from[START_NODE] = null;

while (frontier.Size() > 0)
{
	Node currentNode = frontier.Get(); // 고리 큐의 첫번째 원소를 고름
	
	if(currentNode == DESTINATION_NODE) // 목적지를 진작에 찾았다면
		break; // 조기 이탈!
		
	for (Node neighbor in currentNode.Neighbors()) // 고른 노드의 이웃 노드 중에서
	{
		if (reached.Find(neighbor) == false) // 도착하지 않은 노드가 있다면
		{
			frontier.Add(neighbor); // 그 노드에서 부터 다시 이웃 탐색 하기 위해 고리에 추가
			came_from[neighbor] = currentNode;  // 그 노드에 도착! 경로를 저장함
		}	
	}		
}	
```
<코드 1>
지금 소개된 조기 이탈의 사용법은 가장 대표적이고 간단한 용법이다. 조기 이탈은 이탈하는 조건에 따라 무궁무진하게 활용 될 수 있다. 예를 들어 시작 노드에서 총 n노드를 이동할 수 있는 방법의 경우의 수를 구할 수도 있고, n개의 목적지를 찾을 수도 있다.
